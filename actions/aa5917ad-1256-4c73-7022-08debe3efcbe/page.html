QuickerRpc

Quicker 插件 + 命令行客户端：通过命名管道与 StreamJsonRpc，让外部 qkrpc.exe 调用 Quicker 内部插件能力。

主要用途

  • 外部命令调用 Quicker：CLI 连接动作内加载的插件，由插件在 Quicker 进程内执行实际操作
  • 更新分享动作：当前主要用于调用 ActionEditMgr.UpdateSharedActionAsync 更新已分享动作
  • JSON 输出：命令行支持 --json，便于脚本、自动化工具和 AI Agent 解析结果
  • 本地 RPC 桥接：默认管道名为 QuickerRpc_Server_QRPC2026

架构说明

本动作负责在 Quicker 中加载 QuickerRpc.Plugin，插件启动后监听本机命名管道；外部 qkrpc.exe 作为客户端连接并发起 RPC 调用。

简化流程：Quicker 插件 DLL → 命名管道 / StreamJsonRpc → qkrpc.exe。

CLI 示例

  • 检查插件是否在线：qkrpc ping --json
  • 更新分享动作：qkrpc action update --id <sharedActionGuid> --changelog "说明" --json
  • 也可使用 code 参数：qkrpc action update --code <sharedActionGuid> --json

使用说明

  1. 安装动作后，先按动作内配置加载 QuickerRpc.Plugin 插件依赖
  2. 启动插件服务后，在终端执行 qkrpc ping --json 确认连接正常
  3. 需要自动化更新分享动作时，再执行 qkrpc action update 相关命令

qkrpc.exe 为自包含 CLI，发布脚本会输出到 publish/cli 并可追加到用户 PATH。

环境要求

  • Windows + Quicker 桌面端
  • 动作内需加载 QuickerRpc.Plugin 及 QuickerRpc.Contracts 依赖
  • 外部命令行需要可访问 qkrpc.exe
