动作设计器 2

在 Quicker 内以网页设计器编辑组合动作的步骤与变量：拖拽排序、分支结构、子程序调用、撤销重做，改完后保存回 Quicker，与面板中的动作数据同步。

主要功能

  • 步骤可视化：列表拖拽/插入、分支与占位、按类型搜索插入、复制粘贴步骤 JSON
  • 步骤编辑弹窗：单步参数编辑；关闭时若有未保存修改会提示确认，避免误丢
  • 变量管理：维护动作变量，可高亮引用某变量的步骤（顶栏快捷键说明）
  • 变量批量编辑：多选变量后统一勾选「保存状态 / 输入输出 / 保护变量」等高级项（见下方截图）
  • 子程序侧栏：浏览内部 / 公共 / 网络子程序；网络子程序可搜索标题、说明、标签，拖入步骤列表插入调用
  • 网络共享子程序：搜索 Quicker 分享库；步骤中 @@ 绑定显示可读名称；编辑「调用子程序」时可加载网络子程序的输入/输出变量
  • 全局子程序：主程序与子程序在多标签中切换编辑
  • 撤销 / 重做：步骤与变量共用历史（Ctrl+Z / Ctrl+Y）
  • 首页与检索：最近编辑的动作、按标题/描述搜索后一键进入设计
  • 保存与冲突：Ctrl+S 或顶栏保存到 Quicker；若与 Quicker 侧已有改动会提示是否覆盖

使用说明

  1. 在 Quicker 中安装并触发本动作，打开动作设计器（需配合设计器宿主与 WebView2 环境）
  2. 在首页通过搜索或最近编辑进入要设计的动作；点击左上角图标可返回主界面
  3. 在步骤区维护步骤列表，在弹窗中配置参数；侧栏选用内部/公共/网络子程序并拖入步骤
  4. 在变量区维护变量，可多选后批量编辑高级选项
  5. 完成后保存到 Quicker；离开页面前请处理「未保存」提示

顶栏标注为「Web 预览版 · 动作与变量设计器」；功能持续迭代，欢迎讨论区反馈。

环境要求

  • Windows 10 及以上（推荐 Windows 11）
  • Quicker 1.45.0 及以上
  • .NET 10 桌面运行时与 ASP.NET Core 运行时（见下方「.NET 10 运行时」）
  • WebView2 运行时（多数 Win10/11 已自带或可自动安装）
  • 设计器宿主（WPF + WebView2）与 Quicker 内插件配合使用

.NET 10 运行时（设计器宿主必需）

动作设计器由 WPF + WebView2 宿主承载，本地服务基于 .NET 10，且同时使用 WPF 与 ASP.NET Core。 请安装 Windows Desktop Runtime 10.x 与 ASP.NET Core Runtime 10.x 两项共享运行时； 仅安装其中之一时，宿主进程仍可能无法启动。

与 IntelliTools 内「.NET 10 安装引导」一致；版本号随官方更新，下列直链以 10.0.8 为例，其它版本请在 .NET 10 官方下载页 选择对应架构。

Windows Desktop Runtime windowsdesktop-runtime-10.0.8-win-x64.exe windowsdesktop-runtime-10.0.8-win-arm64.exe

命令行安装（winget）

在提升权限的命令提示符或 PowerShell 中执行（与 IntelliTools 自动安装相同）：

  • winget install Microsoft.DotNet.AspNetCore.10 --accept-package-agreements --accept-source-agreements
  • winget install Microsoft.DotNet.DesktopRuntime.10 --accept-package-agreements --accept-source-agreements

若 winget 不可用，请使用上表直链或官方下载页手动安装。

常见问题

  • 打不开设计器或白屏？
    先按上方「.NET 10 运行时」安装 Desktop 与 ASP.NET Core 两项运行时，并更新 WebView2；重启 Quicker 与设计器宿主后再试。
  • 保存时提示冲突？
    表示 Quicker 侧动作已被其它途径修改，请确认是否覆盖或先在 Quicker 中同步后再保存。
  • 网络子程序搜不到？
    检查网络与 Quicker 版本；v0.3.4 起已改进分享库搜索防抖与兼容性，建议更新到最新修订版。
