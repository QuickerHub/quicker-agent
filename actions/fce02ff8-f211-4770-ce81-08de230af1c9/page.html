动作管理

在独立 WPF 窗口中浏览、筛选、排序面板里的组合动作与公共子程序：支持多规则视图、自定义分组、引用子程序联动筛选，以及运行调试、用量统计与数据体积分析。

本版亮点（v2 规则）

相较早期版本，右侧规则区可保存多条筛选/排序方案：每条规则独立配置「筛选」与「排序」，点击切换即可在不同视角间快速切换，无需反复改条件。

  • 规则：添加 / 删除 / 重命名（双击或右键），当前规则决定列表展示
  • 排序：按编辑时间、分享时间、名称、大小、使用次数、进程等升序或降序
  • 筛选：动作类型、体积区间、来源（自写/安装）、已分享、网址动作、自动更新、使用次数、关联进程等

界面布局

  • 左侧：公共子程序列表，可搜索、排序；支持「引用」某子程序后，右侧动作列表仅显示引用它的动作
  • 右侧：动作表格（图标、标题、描述、大小、进程、时间、使用次数等列可调整顺序）
  • 底部分组：自定义分组标签，将动作归类；删除分组不会删除动作本身
  • 状态栏：当前筛选/排序摘要、引用子程序名、列表条数；快捷入口见下文

主要功能

  • 多选与右键：继承 Quicker 原生动作菜单（编辑、分享等），并扩展分组、复制动作 ID / Data、删除等
  • 运行与调试：列表「运行」弹出层支持直接运行、调试运行及填写运行参数
  • 详细信息：「查看」弹出属性面板，双击文本可复制字段内容
  • 刷新：F5 或顶栏「刷新」，在 Quicker 中新增/修改动作后同步列表
  • 用量统计：查看动作使用次数等统计信息
  • 数据优化分析：对比动作 Data 优化前后体积，便于瘦身（单条或批量入口）
  • 动作回收站：管理已删除动作的恢复与清理

使用说明

  1. 在 Quicker 面板安装本动作并触发，打开动作管理窗口
  2. 顶栏搜索框按标题、描述、进程名快速过滤；需要按子程序反查引用时，用「引用」选择公共子程序
  3. 右侧添加规则并配置筛选/排序；切换规则即可切换列表视角
  4. 底部分组可新建、重命名、删除；右键动作可「添加到分组」或从当前分组移除
  5. 双击动作行或右键菜单进入 Quicker 常规编辑流程；改完后记得 F5 刷新

当前插件版本 1.0.54（以动作包为准）。

环境要求

  • Windows 10 及以上
  • .NET Framework 4.7.2 或更高
  • Quicker 1.39.0 及以上（建议 1.45.0+）

常见问题

  • 列表为空或缺少新动作？
    在 Quicker 中保存动作后，回到本窗口点击刷新（F5）。
  • 规则切换后列表不对？
    确认当前选中的是目标规则；每条规则的筛选、排序互不影响，可删除无用规则后重建。
  • 引用子程序后右侧无动作？
    表示当前没有动作引用该公共子程序；点击「引用」区域的取消即可恢复全部列表。
  • 删除分组会丢动作吗？
    不会。仅移除分组关系，动作仍保留在 Quicker 中。
