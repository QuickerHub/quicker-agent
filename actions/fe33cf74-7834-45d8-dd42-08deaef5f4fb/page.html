剪贴板检视器

Windows 剪贴板格式检视工具：枚举当前数据对象上的全部格式名，按格式查看文本、十六进制或图像预览。除系统剪贴板外，还支持将拖拽数据直接丢进窗口分析（弥补 FreeClipboardViewer 等工具不便查看拖拽内容的短板）。

界面说明

  • 左侧「格式」：列出当前数据对象报告的全部格式名（按字母排序）
  • 右侧「数据」：选中格式后展示内容；图像格式会在上方显示缩略预览
  • 顶栏：「从剪贴板加载」重新读取系统剪贴板；「来源」显示来自剪贴板或拖拽
  • 底栏状态：加载结果、复制格式名等操作提示

主要功能

  • 剪贴板加载：启动时自动读取；也可随时点击顶栏按钮刷新
  • 拖拽分析：将文件或其它支持拖放的数据拖到窗口任意位置，即可像剪贴板一样检视各格式（顶栏显示「来源: 拖拽」）
  • 多类型预览：纯文本直接显示；FileDrop 显示路径列表；二进制以十六进制转储展示（过大时截断，默认最多约 64KB）
  • 图像预览：识别 BitmapSource、字节流等图像载荷，上方显示预览并附尺寸/像素格式说明
  • 复制格式名：在左侧格式项上右键「复制」，便于写 Quicker 步骤或调试自定义格式 ID

使用说明

  1. 在 Quicker 中触发本动作，打开剪贴板检视器窗口
  2. 需要查看当前剪贴板：复制目标内容后，点击「从剪贴板加载」（或重新触发动作）
  3. 需要查看拖拽数据：从资源管理器、浏览器或 Quicker 面板等拖入窗口，勿先依赖剪贴板中转
  4. 在左侧选中感兴趣的格式（如 Unicode Text、HTML Format、应用私有格式），右侧查看解码结果
  5. 开发动作或排查格式兼容性时，可右键复制格式名到剪贴板备用

当前版本 1.3.0（以动作包为准）。

适用场景

  • 确认某应用复制/拖放时提供了哪些剪贴板格式
  • 对比 Unicode、HTML、RTF 等格式下的实际载荷差异
  • 查看图片、文件列表或自定义二进制块的大致内容
  • 编写 Quicker「获取剪贴板」相关步骤前，先摸清格式名与数据结构

环境要求

  • Windows 10 及以上
  • .NET Framework 4.7.2 或更高
  • Quicker 1.39.0 及以上（建议 1.45.0+）

常见问题

  • 格式列表为空？
    当前剪贴板可能无数据，或应用未暴露可读格式；先复制/拖入有效内容后再加载。
  • 某格式显示 GetData 失败？
    部分格式仅对「所有者」可读，或需要特定线程/延迟，属应用限制，可换源应用重试。
  • 二进制只有十六进制？
    非文本且无法解码为图像时，会以十六进制转储显示；超长内容会自动截断。
  • 与 FreeClipboardViewer 的区别？
    本工具侧重拖拽数据与 Quicker 联动；若只需查看常规剪贴板，二者可互补使用。
